01 Project Overview

BUMES (Boston University Manufacturing Execution Software) is a fully automated manufacturing cell that produces a two-part cord organizer, the Cordganizer, from polycarbonate and HDPE stock. The original system took ~24 minutes per cycle to produce two assemblies, with only 40% of that time adding value.

This project had three interlocked goals: redesign the Cordganizer CAD/CAM for faster machining, restructure the robot programs for parallel operation, and build an OpenCV-based vision pipeline so a third robot (Rosie) could sort finished parts by color without human input.

Aim 1: CAD/CAM Redesign

Strip unnecessary operations, tool changes, and machining distance. Target: halve milling time per part.

Aim 2: Process Parallelization

Restructure UR programs and task scheduling for alternate machining across two CNCs, eliminating idle time.

Aim 3: Vision Sorting

Integrate OpenCV color classification with robotic pick-and-place to sort finished parts into bins.

02 The Manufacturing Cell

The BUMES cell spans multiple stations: two UR arms (Mary at Station 3, Edie at Station 4), two Haas CNC mills (Cayenne and Paprika), a conveyor, gravity feeders, squaring fixtures, and Rosie at the vision/sorting station.

03 Aim 1: CAD/CAM Redesign

The original design included unnecessary facing passes, redundant hole operations, and four chamfers per lid. That is machine time that didn't contribute to fit or function. The redesigned CAD reduced operations to pocket milling with two precision holes and two chamfers for the lid, and a single roughing pass with deep pocket milling for the body.

Old lid (4 holes + 4 chamfers) → New lid (pocket + 2 holes + 2 chamfers)

04 Aim 2: BUMES Script Optimization

The original cycle ran Mary and Edie sequentially, meaning one robot stayed idle while the other machined. The redesigned orchestration introduced alternate machining: while Mary finishes a body on Cayenne, Edie begins a lid on Paprika. Steady-state overlap doubled effective throughput.

13 individual UR programs were written (7 per robot), using a letter-and-number naming convention to eliminate confusion. The cycle now produces 4 cordganizers per 24.6-minute run, versus 2 in 24 minutes before.

Bottleneck Analysis

  • Body milling on Paprika: 230 s at 100% utilization - confirmed bottleneck
  • Lid milling on Cayenne: 125 s at 54% utilization
  • Load operations: 70–75 s at ~30% utilization
  • Assembly step: 90 s at 39% utilization
  • Alternate machining reduces effective cycle from 11.9 → 7.2 min/assembly

05 Aim 3: Vision-Based Sorting

When the original inspection camera failed three days before the final demo, I rebuilt the vision pipeline from scratch using OpenCV. The new system uses edge detection and contour analysis to find assembled cordganizers in the camera's ROI, classifies them as Blue or White based on average BGR values, and signals Rosie to route each part to the correct bin.

def classify_color(bgr):
    b, g, r = bgr
    if b < 180 and g < 120 and r < 130:
        return "Blue"
    elif r > 120 and g > 120 and b > 120:
        return "White"
    else:
        return "Unknown"

# Region of interest
roi = frame[50:360, 140:450]

# Edge detection
gray = cv2.cvtColor(roi, COLOR_BGR2GRAY)
blurred = cv2.GaussianBlur(gray, (5,5), 0)
edges = cv2.Canny(blurred, 30, 100)

# Find & filter contours
contours, _ = cv2.findContours(
    edges, RETR_EXTERNAL, CHAIN_APPROX_SIMPLE)
for cnt in contours:
    if cv2.contourArea(cnt) > 1000:
        b, g, r = cv2.mean(obj_roi)[:3]
        return classify_color((b, g, r))

08 Limitations & Future Work

Current Limitations

  • Camera sensitivity to ambient lighting reduced classification reliability
  • No built-in wait() capability between BUMES operations caused timing issues
  • Vision limited to two colors; no defect geometry detection
  • Pose waypoints manually defined instead of being adaptive to part variation

Future Work

  • Optimize CAM for snap-fit-only assembly; further reduce milling times
  • Eliminate burring through finishing passes or reduced feed
  • Expand vision: size classification, hole detection, surface defect ID
  • Integrate SPC tracking per-part across full production run
  • Implement adaptive wait() synchronization between UR scripts
